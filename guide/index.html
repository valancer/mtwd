[개발시 유의할점]
  • iframe : title=""제공.

    * 빈 iframe일경우 : title="내용없음" 제공.

  • input, select, textarea의 경우 label과 연결 : <label for="name">이름</label><input type="text" id="name" style="width:20px">

    * 적절한 label을 연결할 수 없을 경우 title=""제공 : <input type="text" title="검색어입력">

  • <img src="">일 경우 : alt="이미지와 동일한 내용입력" 제공.
  • <inupt type="image"> 일 경우 : alt="이미지와 동일한 내용입력" 제공.

    * 의미없는 이미지일 경우는 빈 alt="" 제공.

  • 새창이 떠야할 경우 새창임을 알림 : target="_blank" 일경우 title="새창이동" 제공.
  • 키보드 접근성 : 모든 기능은 키보드만으로도 사용할 수 있어야 한다.
    1. onkeypress, onkeydown, onkeyup은 onclick가 동일한 기능을 수행하므로 동시에 사용하지 않아야 함.
    2. onkeypress는 일부 브라우저에서 엔터키가 아닌 다른 키에 반응하므로 가급적 onclick로 사용 권장.
    3. onmouseover와 onfocus를 같이 사용.
    4. onmouseout과 onblur를 같이 사용.
    5. onfocus="this.blur();"는 키보드의 접근을 막음으로 사용을 하지 말아야 함.

모바일 접근성
  • 모바일 디바이스 음성 리더기마다 코딩의 표현방식이 달라 접근성이 떨어질 수 있음.
    이를 방지하기 위해 아래와 같은 방식을 적용하여 모바일 웹접근성을 높임.
1. 링크 텍스트에 title값 제공
  • Pc Web : <a href="#" title="새창열림" targer="_blank">text</a>
  • Mobile Web : 링크 텍스트와 title이 동시에 사용될 경우에는 title자체를 숨김처리하여 제공
text새창열림
2. 링크 이미지에 title값과 alt값 제공
  • Pc Web : <a href="#" title="새창열림" targer="_blank"><img src="이미지경로" alt="테스트중"></a>
  • Mobile Web : 부가정보 추가시에는 숨김텍스트 및 alt속성으로 대체 텍스트와 부가정보를 함께 제공

새창열림
3. input에 title과 placeholder 동시제공(value값 동일)
  • Pc Web : <input type="text" title="테스트" placeholder="모바일내용내용">
  • Mobile Web :
    • label 및 aria-labelledby 속성으로 id값을 연결 하여 숨김처리를 해주거나 title값에 placeholder을 대신할 수 있는 내용을 제공.
    • 안드로이드 경우 서식의 레이블을 여러방법으로 제공하여도 우선권이 높은 한가지만 인식함.
    • aria-labelledby 속성은 가장 우선권이 높은 속성이며 속성과 관련된 내용은 http://nuli.navercorp.com/sharing/blog/post/1132700참고

Naming 규칙
[기본규칙]
  • naming은 영문 소문자, 숫자, 언더스코어( _ )로 작성한다.
  • naming의 첫 시작은 영문 소문자로만 시작한다.
  • id 및 class는 단어조합시 카멜표기법과 언더스코어( _ )방식을 사용한다.
    예) tabType1, tab_type1
  • 단어조합시 [형태+의미+상태]의 형태로 작성한다.
  • id는 문서 내 한 번만 사용되며, id/class naming은 가급적 같지 않게 naming 한다.
  • 약속어를 기본으로 작성하나, 없으면 종류와 특성을 나타내도록 naming한다.

[naming 확장]
  • naming 확장시 기존의 class 끝에 숫자를 붙여 추가한다.
    예) tabType1, tab_type1

[약속어]
  • 객체, 이미지, 파일 및 네이밍 시 의미를 일관되게 표현하기 위해 정의.

    약속어
    타이틀 tit_ 작은타이틀 stit_dep1_num/ stit_dep2_num/ stit_dep3_num
    텍스트 txt_ 탭 tab_ 버튼 btn_
    이미지 img_ 배경 bg_ 스프라이트 이미지 sp_
    아이콘 ico_ 불릿 bl_ 라인 line_
    팝업 pop_ 레이어 layer_ 상태변화 _off, _on, _over
    iframe 파일 ifr_ 임시파일 tmp_ include 파일 inc_

[파일 naming 조합]
  • HTML파일은 화면ID로 작업한다. 예) 화면ID.html
CSS
[코드작성 규칙]
  • 모든 속성은 영문 소문자로만 작성한다.
  • @charset 선언시에만 큰따옴표("")를 사용하고 그외의 속성 파라미터값에는 작은 따옴표('')를 사용한다.
  • background의 url 속성의 파라미터값에는 따옴표를 사용하지 않는다.
  • 마지막 속성의 세미콜론(;)은 삭제한다.
  • css 코드는 작성할때는 들여쓰기를 하지 않는다.
    단, 중괄호"{}"가 중첩되는 경우는 예외
  • 속성간에는 공백, 줄내림을 하지 않으며 모든 속성을 붙여쓰기 한다.
    단, 중괄호"{}"가 중첩되는 경우는 예외
  • 중괄호 좌우 공백은 제거한다.
  • 의미있는 객체를 구분하기 위한 빈 줄은 한줄을 초과하지 않게 한다.
  • CSS 최적화를 위해 약식 속성을 이용하여 css 속성이 반복되지 않게 한다.
    예) font, margin, padding, background, border
  • 약식 속성을 이용하되 속성값이 하나만 있는 경우는 약식 속성을 이용하지 않는다.
    예) padding-left:20px
  • 최적화를 위해 '0'뒤의 단위는 생략한다.
    예) padding-left:0
  • 반복되는 색상값은 축약해서 사용한다.
    예) #555555 → #555
  • 배경이미지 위치값은 문자식 표현(left, top, center)은 숫자식(0 0,100% 50%)으로 선언한다.

[속성 선언 순서]
  • 속성을 선언할 때는 가급적 그 쓰임새가 레이아웃과 관련이 큰 것에서 시작하여 레이아웃과 무관한 것 순서로 선언한다.

    속성 선언 순서
    순서 의미 대표되는 속성
    1 Layout display, visibility, overflow, float, clear, position, top, right, bottom, left, z-index
    2 Box width, height, margin, padding, border
    3 배경 background
    4 Font font, color, letter-spacing, text-align, text-decoration, text-indent, vertical-align, white-space
    5 기타 기타(위에 언급하지 않은 나머지 속성들)
  • 밴더속성과 핵속성은 해당 속성 뒤에 선언한다.
    예) .btn{border:1px solid #f60;*border:2px solid #f60;border-radius:2px;-webkit-border-radius:2px}

[단위]
  • px을 사용한다.

[z-index]
  • z-index 값은 최소 10부터 시작하며 최고 9999이며, 10 단위로 증감한다. 단, 중간에 변수가 생기면 예외로 1단위 값을 지정할 수 있다.
